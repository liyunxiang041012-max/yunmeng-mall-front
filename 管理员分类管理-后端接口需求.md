## 1. 数据模型

```sql
categories (
  id          BIGINT      PRIMARY KEY AUTO_INCREMENT,
  name        VARCHAR(50) NOT NULL,          -- 分类名称
  parent_id   BIGINT      DEFAULT 0,         -- 0 = 一级分类, 非0 = 父分类ID
  sort        INT         DEFAULT 0,         -- 排序（可选）
  deleted     TINYINT     DEFAULT 0,         -- 软删除: 0=正常, 1=已删除
  create_time DATETIME    DEFAULT NOW(),
  update_time DATETIME    DEFAULT NOW() ON UPDATE NOW()
)
```

**约束：**
- 支持多级分类（一级 → 二级 → 三级 → ...），不限制层级深度
- `parent_id = 0` 是一级分类
- 同级同父下 `name` 不重复

---

## 2. 接口清单

| 方法 | 路径（示例） | 说明 |
|---|---|---|
| GET | `/it/categories/admin/all` | 获取全部未删除分类，平铺列表 |
| POST | `/it/categories/admin` | 新增分类 |
| PUT | `/it/categories/admin/{id}` | 编辑分类 |
| DELETE | `/it/categories/admin/{id}` | 删除分类（级联软删除子分类） |

---

## 3. 接口详情

### 3.1 获取全部分类

```
GET /it/categories/admin/all

Response:
{
  "code": 200,
  "message": "success",
  "data": [
    { "id": 1, "name": "图书", "parentId": 0 },
    { "id": 2, "name": "小说", "parentId": 1 },
    { "id": 3, "name": "科技", "parentId": 1 }
  ]
}
```

**规则：**
- 只返回 `deleted = 0` 的数据
- 平铺列表（非树形），前端自行构建父子关系
- `parentId = 0` 即一级分类

---

### 3.2 新增分类

```
POST /it/categories/admin
Content-Type: application/json

Request:
{
  "name": "服装",
  "parentId": 0
}

Response:
{
  "code": 200,
  "message": "success",
  "data": { "id": 5, "name": "服装", "parentId": 0 }
}
```

**校验规则：**
| 条件 | 错误提示 |
|---|---|
| name 为空 | 分类名称不能为空 |
| name 长度 < 2 或 > 50 | 分类名称长度需在 2-50 字符之间 |
| parentId 非 0 且父分类不存在 | 父分类不存在 |
| 同级同父下 name 已存在 | 该分类名称已存在 |

---

### 3.3 编辑分类

```
PUT /it/categories/admin/{id}
Content-Type: application/json

Request:
{
  "name": "新名称",
  "parentId": 0
}

Response:
{
  "code": 200,
  "message": "success",
  "data": { "id": 5, "name": "新名称", "parentId": 0 }
}
```

**校验规则：**
- 同上新增校验
- 额外：不能将分类的 `parentId` 设为自己或其子孙分类（防止循环引用）

---

### 3.4 删除分类

```
DELETE /it/categories/admin/{id}

Response:
{
  "code": 200,
  "message": "success",
  "data": null
}
```

**规则：**
- 软删除：设置 `deleted = 1`
- 如果该分类有子孙分类，级联软删除所有子孙分类
- 如果分类已绑定商品，由你决定是否拦截（前端不做限制）

---

## 4. 响应码约定

遵循项目已有的 `code: 200` 成功规约：

```json
// 成功
{ "code": 200, "message": "success", "data": {...} }

// 失败
{ "code": 400, "message": "分类名称已存在", "data": null }
```
