#### 5. 订单管理模块（全新模块）

**后端需要提供的接口**：

##### 5.1 订单分页列表

```
GET /us/order/admin/page
权限: role = 2

请求参数:
  page         int     页码
  size         int     每页条数（建议20）
  keyword      string  订单号/用户名/商品名模糊搜索
  status       int     订单状态筛选（可选，不传=全部）
                       0=待付款 1=已付款 2=已发货 3=已完成 4=已取消
  beginTime    string  下单起始时间 yyyy-MM-dd HH:mm:ss
  endTime      string  下单截止时间

Response:
{
  "code": 200,
  "data": {
    "total": 3842,
    "pages": 193,
    "list": [
      {
        "id": 1,
        "orderNo": "YM20260617001",       // 订单号
        "userId": 1024,                    // 用户ID
        "userName": "张三",                // 用户名（脱敏处理为 张*三）
        "userPhone": "138****1234",        // 用户手机号
        "shopName": "云梦自营",            // 店铺名
        "totalAmount": 39800,              // 订单总金额（分）
        "discountAmount": 5000,            // 优惠金额（分）
        "actualAmount": 34800,             // 实付金额（分）
        "status": 1,                       // 订单状态 0待付款/1已付款/2已发货/3已完成/4已取消
        "statusText": "已付款",            // 状态文本
        "payTime": "2026-06-17 14:30:00",  // 支付时间
        "createTime": "2026-06-17 14:25:00" // 下单时间
      }
    ]
  }
}
```

#### 8. 商家入驻审核

已有商家管理列表，但缺少「审核入驻申请」的接口。如果商家注册需要管理员审批，需要：

```
GET  /it/shop/admin/pending    待审核商家列表
PUT  /it/shop/admin/{id}/approve   审核通过
PUT  /it/shop/admin/{id}/reject    审核驳回（含原因）
```
