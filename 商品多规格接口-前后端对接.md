# 商品多规格（SKU）前后端对接文档

> 基于数据库 `item_sku` 表结构设计，前端已完成 ShopProducts.vue 改造。

---

## 一、接口变更

### `POST /it/shop/item` — 新增商品

**无规格时（兼容现有，不变）：**
```json
{
  "name": "纯棉T恤",
  "price": 9900,
  "stock": 100,
  "image": "https://xxx.oss.com/xxx.jpg",
  "categoryId": 3,
  "brandId": 1
}
```

**有规格时（新增字段）：**
```json
{
  "name": "纯棉T恤",
  "image": "https://xxx.oss.com/xxx.jpg",
  "categoryId": 3,
  "brandId": 1,
  "specs": [
    { "specName": "颜色", "values": ["白色", "黑色"] },
    { "specName": "尺码", "values": ["S", "M", "L"] }
  ],
  "skus": [
    { "specData": {"颜色":"白色","尺码":"S"}, "price": 9900, "stock": 200 },
    { "specData": {"颜色":"白色","尺码":"M"}, "price": 9900, "stock": 200 },
    { "specData": {"颜色":"黑色","尺码":"M"}, "price": 9900, "stock": 200 },
    { "specData": {"颜色":"黑色","尺码":"L"}, "price": 9900, "stock": 200 }
  ]
}
```

> **判断规则**：后端收到 `specs` 且 `specs.length > 0` → 按多规格处理，否则按单规格处理。
>
> ⚠️ **关键**：多规格模式下，请求体 **不传** 顶层 `price` / `stock`，商品价格自动取所有 SKU 中的最低价，库存自动汇总。

### `PUT /it/shop/item/{itemId}` — 编辑商品

同上，支持 `specs` + `skus` 字段。编辑时前端会回传完整的规格和 SKU 数据，后端全量替换即可。

### `GET /it/shop/item/page` — 商家商品列表

列表接口也需要返回 `specs` + `skus` 字段（编辑时回显用）。
如果担心列表数据量过大，可以只返回 `specs`，SKU 在详情接口再查。

### `GET /it/items/{id}` — 商品详情

**需新增返回字段**（Product.vue 已在读）：

```json
{
  "id": 3,
  "name": "纯棉T恤",
  "price": 9900,
  "mainImage": "https://...",
  "images": ["https://..."],
  "shopId": 10,
  "shopName": "云梦服饰",

  "specs": [
    {
      "specName": "颜色",
      "values": [
        { "value": "白色", "stock": 200 },
        { "value": "黑色", "stock": 200 }
      ]
    },
    {
      "specName": "尺码",
      "values": [
        { "value": "S", "stock": 200 },
        { "value": "M", "stock": 200 },
        { "value": "L", "stock": 200 }
      ]
    }
  ],

  "skus": [
    {
      "id": 7,
      "name": "纯棉T恤 白色 S",
      "price": 9900,
      "stock": 200,
      "image": null,
      "specData": { "尺码": "S", "颜色": "白色" }
    }
  ]
}
```

## 二、字段说明

### `specs[]` — 规格组
| 字段 | 类型 | 说明 |
|------|------|------|
| `specName` | string | 规格名，如"颜色"、"尺码" |
| `values` | string[] | 规格值列表（新增时） |
| `values` | object[] | `[{value, stock}]`（详情返回时，stock 为该规格值下有多少库存） |

### `skus[]` — SKU 明细
| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `specData` | object | ✅ | JSON 对象，规格键值对，如 `{"颜色":"白","尺码":"S"}` |
| `price` | int | ✅ | 价格，**单位分**（前端 ×100 / ÷100） |
| `stock` | int | ✅ | 库存 |
| `image` | string | | SKU 独立图片（当前前端不传，预留） |

### `sku_name` 生成规则
数据库字段 `sku_name` 无需前端传，后端根据 `商品名 + specData.values` 拼接生成，如：`"纯棉T恤 白色 S"`。

### 多规格时商品顶层价格
由后端自动计算 = `MIN(skus[].price)`，库存 = `SUM(skus[].stock)`。

## 三、数据库 `item_sku` 参考字段

```
id          BIGINT    主键
item_id     BIGINT    关联 SPU id
sku_name    VARCHAR   如 "小米14 Pro 黑色 128G"
price       BIGINT    价格（分）
stock       INT       库存
image       VARCHAR   SKU图片（可为NULL）
spec_data   JSON      如 {"颜色":"黑色","存储":"128G"}
status      TINYINT   1=正常 0=禁用
deleted     TINYINT   软删除
```

## 四、前端取值路径

```js
// 商品详情
const d = response.data
itemDetail.value  = d
skuList.value     = d.skus        // ← 新增
specGroups.value  = d.specs       // ← 新增

// SKU 选中匹配
const currentSku = skuList.value.find(sku =>
  Object.entries(selectedSpecs).every(([k, v]) => sku.specData[k] === v)
)
const currentPrice = currentSku?.price ?? itemDetail.value?.price
```
