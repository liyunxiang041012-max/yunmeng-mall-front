## 需要后端补充/确认的接口

### 1. `GET /it/shop/stats` — 仪表盘数据

**当前返回字段**: `totalProducts`, `activeProducts`, `shopId`, `shopName`, `logo`, `status`

**需要新增字段**（前端概览页有 4 张数据卡片，目前只能填 2 张）:

```json
{
  "code": 200,
  "data": {
    "shopId": 4,
    "shopName": "云梦优选",
    "logo": "https://oss.xxx/xxx.jpg",
    "status": 1,

    "totalProducts": 15,
    "activeProducts": 10,

    "todaySales": 1285000,
    "todayOrders": 86,
    "totalRevenue": 52345600,
    "totalOrders": 320,
    "views": 3482
  }
}
```

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| todaySales | Long | 分 | 今日销售额 |
| todayOrders | Integer | — | 今日订单数 |
| totalRevenue | Long | 分 | 累计收入 |
| totalOrders | Integer | — | 累计订单数 |
| views | Integer | — | 店铺浏览量 |

> **优先级**: 中。目前前端概览页只能填 2 张卡片（全部商品、上架中），其余 2 张显示 `...`。补上这些字段后卡片全满。

### 2. `GET /it/shop/orders` — 订单列表

**当前返回字段**: `id`, `shopId`, `userId`, `totalAmount`, `payAmount`, `discountAmount`, `status`, `createTime`, `updateTime`, `payTime`, `expireTime`

**需要确认/补充的字段**:

```json
{
  "id": "1718726400000123456",
  "userId": 100,
  "shopId": 4,
  "totalAmount": 19900,
  "payAmount": 0,
  "discountAmount": 0,
  "status": "PENDING_PAYMENT",
  "createTime": "2026-06-18 10:00:00",
  "updateTime": "2026-06-18 10:00:00",
  "payTime": null,
  "expireTime": "2026-06-18 10:30:00",

  "productName": "手工牛皮托特包",
  "productImage": "https://oss.xxx/item/xxx.jpg",
  "buyerName": "张先生"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| productName | String | 订单中的商品名（至少取第一个商品） |
| productImage | String | 商品图片URL |
| buyerName | String | 买家昵称/手机号 |

> **优先级**: 高。前端订单列表表格显示"商品"和"买家"列，目前无法展示。

**查询参数支持**:

| 参数 | 类型 | 说明 |
|------|------|------|
| page | int | 页码 |
| size | int | 每页条数 |
| status | String | 按状态筛选 (PENDING_PAYMENT / PAID / SHIPPED / COMPLETED / CANCELLED) |
| keyword | String | 关键词搜索（订单号/商品名） |

## 统一规范确认

### 响应格式
所有接口统一返回 `{ "code": 200, "data": ... }`。  
前端 Axios 响应拦截器: `res => res.data`，解包后拿到 `{code, data}` 对象，取 `.data` 获取业务数据。

### 金额单位
所有金额字段（price / amount / totalAmount / payAmount / discountAmount 等）**统一使用 分（Long）**，前端展示时除以 100 并 `.toFixed(2)`。

### 订单状态枚举
使用 **String 枚举**（已对接）:

| 值 | 含义 |
|---|------|
| PENDING_PAYMENT | 待付款 |
| PAID | 已付款/待发货 |
| SHIPPED | 已发货 |
| COMPLETED | 已完成 |
| CANCELLED | 已取消 |

### 时间格式
统一 `yyyy-MM-dd HH:mm:ss` 格式字符串。
