# 后端接口对接文档 — 收藏 & 历史 & 商品筛选

> 本文档描述前端已对接的后端接口，包含请求路径、参数、返回值 JSON 示例，供后端开发参照实现。
> **通用规则**：所有接口返回值统一包裹在 `Result<T>` 结构中：`{ code: 200, data: T, msg: "ok" }`，价格字段单位均为 **分（fen）**。

---

## 1. GET /it/favorites/my — 我的收藏列表

**说明**：返回当前登录用户的收藏商品列表，支持分页。

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 10 |

**返回值**：`Result<PageResult<FavoriteItemVO>>`

```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "records": [
      {
        "id": 1,
        "itemId": 101,
        "name": "极简无线降噪耳机",
        "itemName": "极简无线降噪耳机",
        "mainImage": "/images/item/101/main.jpg",
        "image": "/images/item/101/thumb.jpg",
        "price": 8900,
        "salesCount": 1200,
        "createTime": "2026-05-20 14:30:00"
      }
    ],
    "total": 34,
    "page": 1,
    "size": 10
  }
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 收藏记录 ID（用于取消收藏定位）|
| itemId | Long | 商品 ID（调用 toggle/check 接口时使用）|
| name / itemName | String | 商品名称（两者至少返回一个）|
| mainImage | String | 商品主图（优先使用）|
| image | String | 商品图片（mainImage 为空时的备选）|
| price | Integer | 商品价格，单位 **分** |
| salesCount | Integer | 销量（可选，店铺页展示用）|
| createTime | String | 收藏时间 yyyy-MM-dd HH:mm:ss |

## 2. GET /it/history/my — 我的浏览历史

**说明**：返回当前登录用户的浏览历史列表，支持分页。

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 10 |

**返回值**：`Result<PageResult<HistoryItemVO>>`

```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "records": [
      {
        "id": 1,
        "itemId": 201,
        "name": "轻量碳纤维背包",
        "itemName": "轻量碳纤维背包",
        "mainImage": "/images/item/201/main.jpg",
        "image": "/images/item/201/thumb.jpg",
        "price": 12900,
        "createTime": "2026-05-28 10:15:00",
        "viewTime": "2026-05-28 10:15:00"
      }
    ],
    "total": 210,
    "page": 1,
    "size": 10
  }
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 浏览记录 ID（用于删除单条）|
| itemId | Long | 商品 ID |
| name / itemName | String | 商品名称（两者至少返回一个）|
| mainImage | String | 商品主图（优先使用）|
| image | String | 商品图片（备选）|
| price | Integer | 商品价格，单位 **分** |
| createTime | String | 浏览时间 yyyy-MM-dd HH:mm:ss |
| viewTime | String | 浏览时间（createTime 的别名，两者至少返回一个）|

## 3. DELETE /it/history/{id} — 删除单条浏览记录

**说明**：删除指定 ID 的浏览记录。

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | Long | 浏览记录 ID |

**返回值**：`Result<Void>`

```json
{
  "code": 200,
  "msg": "删除成功",
  "data": null
}
```

**说明**：清空当前登录用户的所有浏览记录。

**返回值**：`Result<Void>`


## 5. GET /it/items/page — 商品分页（已有接口，新增 shopId 过滤）

**说明**：已有商品分页接口，**新增 `shopId` 可选查询参数**，用于按店铺筛选商品。

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 20 |
| shopId | Long | **否** | **新增** — 店铺 ID，传入则只返回该店铺的商品 |
| sort | String | 否 | 排序：`default`(综合) / `sales`(销量) / `price`(价格) / `newest`(最新) |

**返回值**：`Result<PageResult<ItemVO>>`（与现有格式完全一致，仅多支持 shopId 筛选）

```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "records": [
      {
        "id": 301,
        "name": "AI智能降噪蓝牙耳机旗舰版",
        "mainImage": "/images/item/301/main.jpg",
        "image": "/images/item/301/thumb.jpg",
        "price": 59900,
        "originalPrice": 89900,
        "salesCount": 800,
        "shopId": 10,
        "shopName": "云梦数码旗舰店"
      }
    ],
    "total": 56,
    "page": 1,
    "size": 20
  }
}
```

**字段说明**（店铺页用到的字段）：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 商品 ID |
| name | String | 商品名称 |
| mainImage | String | 商品主图 |
| image | String | 商品图片（备选）|
| price | Integer | 当前价格，单位 **分** |
| originalPrice | Integer | 原价，单位 **分**（可选）|
| salesCount | Integer | 销量 |
| shopId | Long | 所属店铺 ID |

## 附录 A：已有接口（本次改动中已对接，供参考）

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/it/favorites/toggle/{itemId}` | 收藏/取消收藏（toggle） |
| GET | `/it/favorites/check/{itemId}` | 检查是否已收藏，返回 `{ favorited: true/false }` |
| POST | `/it/shop/follow/toggle/{shopId}` | 关注/取消关注店铺 |
| GET | `/it/shop/follow/check/{shopId}` | 检查是否已关注，返回 `{ followed: true/false }` |
| GET | `/it/shop/{shopId}` | 获取店铺详情 |
| GET | `/it/items/{id}` | 获取商品详情（需包含 `shopId` 字段） |

## 附录 C：图片路径处理

前端会自动处理图片路径：
- 以 `http` 开头：直接使用
- 以 `/` 开头：拼接后端基地址 `http://localhost:8080`
- 其他相对路径：拼接 `http://localhost:8080/` + 路径

建议后端返回以 `/` 开头的绝对路径，如 `/images/item/101/main.jpg`。
