# 云梦商城 — 完整接口文档（前端对接版）

> 基于代码实际扫描，覆盖所有 32 个 Controller。标注了网关路由、鉴权、响应格式。

---

## 网关架构

```
前端 → http://localhost:8080 → Spring Cloud Gateway (StripPrefix=1)
       /us/**  → ym-user       用户服务
       /it/**  → ym-item       商品/商家服务
       /ca/**  → ym-pay        购物车
       /py/**  → ym-pay        订单/支付
       /pm/**  → ym-promotion  优惠券
       /rm/**  → ym-remark     评论/点赞
       /ai/**  → yunmeng-ai    AI助手
```

**鉴权白名单**（无需 Authorization header）：
- `POST /us/user/register`
- `POST /us/user/login`
- `POST /us/user/shop/login`
- `POST /us/user/admin/login`
- `POST /us/user/sendCode`
- `POST /it/shop/register`

其他接口需要 `Authorization: Bearer <token>`。

**响应格式**：标记为 `✅Result` 的返回 `{code:200, msg:"ok", data:...}`；标记为 `⚠️Raw` 的返回原始对象/列表（这些是Feign内部调用端点，前端直接调用也能用但格式不统一）。

## 附录A：响应格式说明

| 标记 | 含义 | 示例 |
|:--:|------|------|
| ✅Result | 标准 `{code:200, msg:"ok", data:...}` | `{"code":200,"data":{"token":"xxx","role":1}}` |
| ⚠️Raw | Feign内部端点，直接返回对象 | `{"id":1,"name":"店铺","logo":"..."}` |

> 前端 axios 拦截器 `res => res.data` 解包后：Result 接口得到 `{code, msg, data}`；Raw 接口直接得到数据对象本身。
