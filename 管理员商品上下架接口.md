# 管理员商品上下架接口

## 背景

管理员后台（AdminDashboard）需要对已审核通过的商品进行上下架操作。此前前端误调了商家端接口 `/it/shop/item/toggle-status/{id}`，该接口要求 role=1（商家），管理员 role=2 被拦截，提示「只有商家才可以访问」。

现已将前端调用改为管理员专属路径，需后端新增对应接口。

---

## 接口

```
PUT /it/shop/admin/item/toggle-status/{id}
```

### 路径参数

| 参数 | 类型 | 说明 |
|------|------|------|
| id | Long | 商品 ID |

### 请求头

```
Authorization: Bearer <admin_token>
```

### 请求体

无

### 权限

要求 role = 2（管理员）

### 逻辑

与商家端 `PUT /it/shop/item/toggle-status/{id}` 完全一致：

```
status == 0 → 改为 1（上架）
status == 1 → 改为 0（下架）
```

### 响应

```json
{
  "code": 200,
  "message": "操作成功"
}
```

---

## 前端调用位置

- `src/api/admin.js` — `toggleAdminItemStatus(id)`
- `src/views/AdminDashboard.vue` — `handleAdminToggleStatus(p)`，仅对 `auditStatus === 1`（已通过）的商品展示
