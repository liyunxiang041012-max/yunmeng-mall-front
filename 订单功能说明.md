# 订单功能实现说明

## 概述
前端订单功能已经完成,包括创建订单、查看订单详情、支付等完整流程。

## 功能流程

### 1. 购物车结算 (Cart.vue)
- **位置**: `src/views/Cart.vue`
- **功能**: 
  - 选择购物车商品
  - 检测价格变动并提示用户
  - 调用创建订单接口
  - 跳转到支付页面

- **关键代码**:
```javascript
// 创建订单参数格式
const orderDTO = {
  items: [
    { skuId: 1, quantity: 2 },
    { skuId: 2, quantity: 1 }
  ]
}

// 调用接口
const res = await createOrder(orderDTO)
// 返回: 订单ID字符串
```

### 2. 订单详情 (Pay.vue)
- **位置**: `src/views/Pay.vue`
- **功能**:
  - 根据订单ID获取订单详情
  - 显示订单商品信息
  - 选择收货地址
  - 选择支付方式
  - 完成支付

- **订单数据映射**:
```javascript
// 后端 Order 对象 → 前端 payData
payData.value = {
  orderId: orderData.id,                    // 订单ID(订单号)
  orderNo: orderData.id,                    // 订单号
  items: orderData.items || [],             // 订单项列表(如果后端有联表查询)
  subtotal: orderData.totalAmount / 100,    // 订单总金额(分→元)
  couponDiscount: orderData.discountAmount / 100,  // 优惠金额(分→元)
  shippingFee: 0,                           // 运费(后端暂无)
  totalAmount: orderData.payAmount / 100,   // 实付金额(分→元)
  deliveryType: '标准快递'                  // 配送方式(前端默认)
}
```

**重要提示**:
- Order 对象本身**不包含 items 字段**
- 如果后端 `getOrderDetail` 接口有做联表查询,会返回 items
- 如果没有,需要单独调用接口获取订单项列表

### 3. 支付功能
- **支付方式**: 微信支付、支付宝、银行卡
- **支付参数**:
```javascript
const payDTO = {
  orderId: '订单ID',
  payChannel: 'WECHAT',  // WECHAT/ALIPAY/BANK_CARD
  amount: 10000,         // 金额,单位:分
  addressId: 1,          // 收货地址ID
  note: '',              // 订单备注
  couponId: null         // 优惠券ID
}
```

## 金额处理

### 重要提示
- **后端金额单位**: 分 (Long 类型)
- **前端显示单位**: 元
- **所有涉及金额的字段都需要转换**

### 转换规则
```javascript
// 后端 → 前端: 除以 100
const yuan = fen / 100

// 前端 → 后端: 乘以 100
const fen = Math.round(yuan * 100)
```

### 需要转换的字段

#### 1. Order (订单)
```javascript
// 后端返回 (单位: 分)
{
  totalAmount: 10000,      // 订单总金额
  payAmount: 10000,        // 实付金额
  discountAmount: 0        // 优惠金额
}

// 前端转换后 (单位: 元)
payData.value = {
  subtotal: 100.00,        // totalAmount / 100
  totalAmount: 100.00,     // payAmount / 100
  couponDiscount: 0.00     // discountAmount / 100
}
```

#### 2. OrderItem (订单项)
```javascript
// 后端返回 (单位: 分)
{
  price: 5000,             // 商品单价
  quantity: 2              // 数量(不需要转换)
}

// 前端转换后 (单位: 元)
{
  price: 50.00             // price / 100
}
```

#### 3. CartItem (购物车项)
```javascript
// 后端返回 (单位: 分)
{
  price: 5000,             // 实时价格
  snapshotPrice: 4500      // 快照价格
}

// 前端转换后 (单位: 元)
{
  price: 50.00,            // price / 100
  snapshotPrice: 45.00,    // snapshotPrice / 100
  newPrice: 50.00          // 价格变动时的新价格
}
```

### 自动转换逻辑

#### 购物车页面 (Cart.vue)
```javascript
// 购物车商品价格转换
items.forEach(item => {
  // 快照价格转换
  if (item.snapshotPrice) {
    item.snapshotPrice = parseFloat((item.snapshotPrice / 100).toFixed(2))
  }
  
  // 实时价格转换
  if (item.price) {
    item.price = parseFloat((item.price / 100).toFixed(2))
  }
  
  // 价格变动时的新价格
  if (item.priceChanged) {
    item.newPrice = item.price
  }
})
```

#### 支付页面 (Pay.vue)
```javascript
// 1. 订单金额转换
if (payData.value.subtotal > 0) {
  payData.value.subtotal /= 100
  payData.value.couponDiscount /= 100
  payData.value.shippingFee /= 100
  payData.value.totalAmount /= 100
}

// 2. 订单项价格转换
if (payData.value.items && payData.value.items.length > 0) {
  payData.value.items.forEach(item => {
    if (item.price) {
      item.price = parseFloat((item.price / 100).toFixed(2))
    }
  })
}
```

#### 支付请求 (Pay.vue - handlePay)
```javascript
// 前端 → 后端: 元转换为分
const payDTO = {
  amount: Math.round(payData.value.totalAmount * 100)  // 元 → 分
}
```

## 错误处理

### 统一的错误提示
```javascript
try {
  await createOrder(orderDTO)
} catch (err) {
  const errorMsg = err.response?.data?.message || 
                   err.response?.data?.msg || 
                   err.message || 
                   '订单创建失败,请重试'
  ElMessage.error(errorMsg)
}
```

### 常见错误码
- `400`: 请求参数错误
- `401`: 登录已过期
- `403`: 没有权限
- `404`: 订单不存在
- `500`: 服务器错误

## 注意事项

1. **用户认证**: 
   - 创建订单和支付都需要登录
   - Token 通过请求拦截器自动添加
   
2. **金额单位**:
   - 后端使用分(Long类型)
   - 前端显示使用元
   - 转换规则:
     ```javascript
     // 后端 → 前端: 除以 100
     const yuan = fen / 100
     
     // 前端 → 后端: 乘以 100
     const fen = Math.round(yuan * 100)
     ```

3. **参数验证**:
   - 创建订单时验证 SKU ID 和数量
   - 支付时验证订单ID、地址、支付方式

4. **错误处理**:
   - 所有接口调用都有 try-catch
   - 用户友好的错误提示
   - 详细的控制台日志

5. **Order 对象不包含 items**:
   - Order 实体类只有订单基本信息
   - 订单项(OrderItem)是单独的表
   - **后端需要**: 
     - 方案1: `getOrderDetail` 接口联表查询,返回 Order + items
     - 方案2: 前端先获取 Order,再调用另一个接口获取 OrderItem 列表
   - **前端已兼容**: 如果后端返回 items 则显示,否则为空数组

6. **订单状态**:
   - 使用 OrderStatus 枚举
   - 0: 待付款
   - 1: 已付款
   - 2: 已发货
   - 3: 已完成
   - 4: 已取消
   - 5: 已退款

7. **后端接口建议**:
   ```java
   // 建议 getOrderDetail 返回完整数据
   @GetMapping("/detail/{orderId}")
   public Result<OrderVO> detail(@PathVariable String orderId) {
       Order order = orderService.getOrderDetail(orderId);
       List<OrderItem> items = orderItemService.getByOrderId(orderId);
       
       OrderVO vo = new OrderVO();
       vo.setOrder(order);
       vo.setItems(items);
       return Result.success(vo);
   }
   ```
