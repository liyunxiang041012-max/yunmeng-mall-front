# 优惠券支付对接 + 秒杀服务 — 前后端接口文档

> 前端已完成页面和逻辑，后端按本文档实现接口即可对接。
> 金额单位：**分**（前端展示时 /100 转为元，提交时 *100 转回分）。
> 响应格式：统一 Result<T> = `{ code: 200, data: T, message: "ok" }`，前端 axios 拦截器自动解包。

## 一、优惠券 — 支付页可用券查询

支付页（`/pay`）加载时会查询该订单可用的优惠券，用户选择后抵扣应付金额，提交支付时传入 `couponId`。

### 1.1 查询订单可用的优惠券

```
GET /pm/user-coupons/order-available?orderAmount=9900
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderAmount | int | 是 | 订单应付金额（分），用于过滤达不到门槛的券 |

**响应 data**：数组，每个元素：

```json
[
  {
    "id": 1001,
    "couponId": 1,
    "name": "新人满减券",
    "discountType": "AMOUNT_DISCOUNT",
    "discountValue": 1000,
    "thresholdAmount": 5000,
    "termEndTime": "2026-12-31 23:59:59"
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | long | 用户券 ID（user_coupon 表主键，支付时传这个） |
| couponId | long | 优惠券模板 ID |
| name | string | 优惠券名称 |
| discountType | string | AMOUNT_DISCOUNT=满减, RATE_DISCOUNT=折扣, FREE_SHIPPING=免邮 |
| discountValue | int | 面额（分），RATE_DISCOUNT 时整数表示折扣率（如 85 表示 8.5折） |
| thresholdAmount | int | 使用门槛（分），0=无门槛 |
| termEndTime | string | 有效期截止 |

**前端调用**：`api/coupon.js` → `getOrderAvailableCoupons(orderAmount)`

---

### 1.2 创建支付时传券

`POST /py/pay/create` 的 PayDTO 中 `couponId` 字段改为实际值：

```json
{
  "orderId": 123,
  "payChannel": "WECHAT",
  "amount": 8900,
  "addressId": 5,
  "note": "",
  "couponId": 1001
}
```

| 字段 | 说明 |
|------|------|
| couponId | 用户券 ID（user_coupon 表 ID），未选券时传 null |

**后端需处理**：
- 验证该券属于当前用户、状态为未使用
- 计算实际抵扣金额
- 支付成功后将该券状态标记为已使用

## 二、管理员优惠券管理

管理员后台（AdminDashboard）新增「优惠券管理」tab，支持 CRUD。

### 2.1 管理员查询优惠券列表（分页）

```
GET /pm/admin/coupons/page?pageNo=1&pageSize=100
```

**响应 data**：

```json
{
  "total": 5,
  "pages": 1,
  "records": [
    {
      "id": 1,
      "name": "新人满减券",
      "discountType": "AMOUNT_DISCOUNT",
      "discountValue": 1000,
      "thresholdAmount": 5000,
      "totalQuantity": 1000,
      "maxPerUser": 1,
      "receivedCount": 342,
      "termStartTime": "2026-06-01 00:00:00",
      "termEndTime": "2026-12-31 23:59:59",
      "description": "新用户专享"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | long | 优惠券模板 ID |
| name | string | 名称 |
| discountType | string | AMOUNT_DISCOUNT / RATE_DISCOUNT / FREE_SHIPPING |
| discountValue | int | 面额（分），RATE_DISCOUNT 时如 85 表示 8.5折 |
| thresholdAmount | int | 门槛（分），0=无门槛 |
| totalQuantity | int | 发放总量，null=不限量 |
| maxPerUser | int | 每人限领，null=不限 |
| receivedCount | int | 已领取数量 |
| termStartTime | string | yyyy-MM-dd HH:mm:ss |
| termEndTime | string | yyyy-MM-dd HH:mm:ss |
| description | string | 描述 |

**前端调用**：`api/admin.js` → `getAdminCouponPage(params)`

---

### 2.2 管理员创建优惠券

```
POST /pm/admin/coupons
Content-Type: application/json

{
  "name": "新人满减券",
  "discountType": "AMOUNT_DISCOUNT",
  "discountValue": 1000,
  "thresholdAmount": 5000,
  "totalQuantity": 1000,
  "maxPerUser": 1,
  "termStartTime": "2026-06-01 00:00:00",
  "termEndTime": "2026-12-31 23:59:59",
  "description": "新用户专享"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| name | 是 | 优惠券名称，最长 30 字 |
| discountType | 是 | AMOUNT_DISCOUNT / RATE_DISCOUNT / FREE_SHIPPING |
| discountValue | 满减/折扣时必填 | 面额（分），RATE_DISCOUNT 时为折扣整数值 |
| thresholdAmount | 否 | 门槛（分），0=无门槛 |
| totalQuantity | 否 | 总量，不传=不限 |
| maxPerUser | 否 | 每人限领 |
| termStartTime | 是 | yyyy-MM-dd HH:mm:ss |
| termEndTime | 是 | yyyy-MM-dd HH:mm:ss |
| description | 否 | 描述 |

**前端调用**：`api/admin.js` → `adminCreateCoupon(data)`

---

### 2.3 管理员更新优惠券

```
PUT /pm/admin/coupons/{id}
```

参数同创建接口。**前端调用**：`api/admin.js` → `adminUpdateCoupon(id, data)`

---

### 2.4 管理员删除优惠券

```
DELETE /pm/admin/coupons/{id}
```

**前端调用**：`api/admin.js` → `adminDeleteCoupon(id)`

## 三、秒杀服务

新建独立页面 `/seckill`，展示进行中/即将开始的秒杀活动。

### 3.1 用户端秒杀活动列表

```
GET /seckill/list
```

**响应 data**：

```json
{
  "ongoing": {
    "id": 1,
    "name": "618 数码专场",
    "startTime": "2026-06-24 10:00:00",
    "endTime": "2026-06-24 22:00:00",
    "items": [
      {
        "id": 301,
        "name": "无线降噪耳机",
        "image": "https://xxx.jpg",
        "originalPrice": 29900,
        "seckillPrice": 19900,
        "stock": 50,
        "totalStock": 200,
        "soldCount": 150
      }
    ]
  },
  "upcoming": [
    {
      "id": 2,
      "name": "周末特惠",
      "startTime": "2026-06-25 10:00:00",
      "endTime": "2026-06-25 18:00:00",
      "items": [...]
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| ongoing | object | 当前进行中的活动（null=无） |
| ongoing.id | long | 活动 ID |
| ongoing.name | string | 活动名称 |
| ongoing.startTime | string | 活动开始时间 |
| ongoing.endTime | string | 活动结束时间 |
| ongoing.items | array | 活动商品列表 |
| upcoming | array | 即将开始的活动列表（可多个），结构同 ongoing |
| items[].id | long | 秒杀商品 ID |
| items[].name | string | 商品名 |
| items[].image | string | 商品图片 URL |
| items[].originalPrice | int | 原价（分） |
| items[].seckillPrice | int | 秒杀价（分） |
| items[].stock | int | 剩余库存 |
| items[].totalStock | int | 秒杀总库存 |
| items[].soldCount | int | 已抢数量 |

**前端调用**：`api/seckill.js` → `getSeckillList()`

> 兼容方案：如果后端返回扁平数组 `[{status: "ONGOING", ...}, {status: "UPCOMING", ...}]`，前端也能自行分类。

### 3.2 秒杀抢购下单

```
POST /seckill/order
Content-Type: application/json

{
  "seckillItemId": 301,
  "quantity": 1
}
```

**响应 data**：同普通下单，返回 `{ orderId, expireTime }`。

**前端调用**：`api/seckill.js` → `seckillOrder(data)`
